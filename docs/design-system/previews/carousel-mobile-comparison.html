Carousel — qué cortamos y qué queda en móvil

La confusión habitual: en Figma hay una página «Carousel» con plantillas M3, pero en MVP ya usáis filas horizontales montadas con Cards en Examples. Son cosas distintas.

CUT = fuera de scope (tablet / fullscreen) LATER = en Figma, sin pantalla JTBD aún MVP hoy = LazyRow + Cards
Dos niveles — no mezclar ① Set «Carousel» (página Figma): layouts completos con nombre M3 (Hero, Multi-browse…). ② Carrusel real en producto: sección «Próximos viajes» = LazyRow de OutlinedCard — ya en Cards/Lists/Examples, no necesita el set de la página Carousel para existir.

1 · Lo que se cortó (y por qué)

Política MyOwnTrip: móvil + foldable sí; tablet 10″ / web / inmersivo fullscreen no. Se eliminaron variantes que solo tienen sentido en anchos ≥600dp o experiencias desktop.

No es JTBD bolsillo MVP

No se cortó el concepto de carrusel. Solo plantillas que asumen tablet o UI inmersiva. El backup CS las conserva por si algún día ampliáis scope.
